Data is stored on Compute nodes only.To understand how Redshift distributes data, we need to know some details about compute nodes.Slice is logical partition for disk storage. A query selecting 5 columns out of 100 column table only has to access 5% of the data block space.Each block of data contains values from a single column. Each node has multiple slices which allow parallel access and processing across slices on each node.The number of slices per node depends on the node instance types. This system of slices and nodes achieves two objectives:One key feature of Redshift that influences the compute is the columnar storage of data. The Leader node manages data distribution and query execution across Compute nodes. Understanding this a database developer can write optimal queries avoiding select * as with OLTP databases.So far, data storage and management have shown significant benefits. Redshift can apply specific and appropriate compression on each block increasing the amount of data being processed within the same disk and memory space.

For demonstration purposes, we will use the pre-built official image of Ruby and macOS.The docker run command is used to run a container from an image. To stop a container, use the docker stop command with either the container id or container name. For that, we will use the docker images and docker system df commands to display information regarding disk space usage by the Docker daemon. Here we are using the ruby image to create the container, but you can use your own image. Now you can use that image to create a container to run your application on different machines or even on the cloud platforms (AWS, GCP, Azure, etc.) If you don’t use that after you stop or exit the container, it has to be manually removed if you want to free disk space. From this post, you’ll learn some basic concepts and commands of Docker that will teach you how to mount a directory inside a Docker container.

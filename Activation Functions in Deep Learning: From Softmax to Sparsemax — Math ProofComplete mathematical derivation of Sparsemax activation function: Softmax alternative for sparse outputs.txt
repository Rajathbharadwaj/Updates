One problem that arises from an attempt to combine sparsemax with cross-entropy is that this loss function would now require full-support, i.e only non-zero valued outputs. That is why cross-entropy loss cannot be used for the sparsemax activation function. This means that sparsemax would assign a probability of exactly 0 for some classes, while softmax would instead keep those classes and assign them very small values like 10⁻³. Such behavior is a property that we also want to maintain in the sparsemax framework.However, empirically, this setup becomes infeasible with sparsemax. While the first proof of closed-form solution was determined directly from the original definition of ​sparsemax, the loss function is a question of preference and can take different forms. But since the loss function takes the logarithm of probabilities, if a probability is strictly null, then its log is not defined.

In all the examples below, the memory location (address) gets changed because a new object gets created at different memory location during the operation. The equality operator compares the actual value stored in the object.An object whose internal state (data) can be changed is called a Mutable object. In the function scope, this creates another object lst which also references to the same memory location as lists are mutable. Similarly, an object whose internal state (data) can not be changed is called an Immutable object. The id() function returns the memory location of the object referenced by a variable. This confirms that lists, sets & dictionaries are mutable objects.list exampleset exampledictionary exampleSince numbers, strings, tuples & frozen sets are immutable, internal state (data) can not be changed.

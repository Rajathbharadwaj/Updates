This blog is part 3 of the series Preserving Data Privacy in Deep Learning and focuses on the implementation of federated learning with the non-IID dataset. Also, the synchronization of each client’s model with global weights along with the weighted_mean as an aggregation of technique helps in getting a good accuracy of this model. the basic federated learning with mean, approach 2 (retraining on a global server) outclassed the one used in part 1. This tutorial builds a federated learning model from scratch using PyTorch by converting the balanced CIFAR10 dataset to the non-IID/real-world dataset. This approach of retraining each client’s model with baseline data deals with the problem of non-IID data. The safety of the client’s data is ensured by only sending the updated weights of the model, not the data. Then the local model is trained on the client’s device itself, following which the retraining round takes place on the global server.

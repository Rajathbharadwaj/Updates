A quick application of the filter option in the data category in excel does the trick.Nothing unexpected in the drug column, though the disease column does contain some noise. The drugs and diseases columns were one-hot-encoded (ohe) and all noisy samples in the diseases column were replaced with a single string, representing a missing value. Since I didn’t have much time, I replaced only a few drug names, manually. The drug_approved_by_UIC feature didn’t make much sense as the same drug received multiple approval dates. The patient_id column did not contain any duplicates in both the train and test sets provided by the platform, so it was of no use. But wait, we know that drugs are sometimes known by multiple names. This resulted in a large, sparse feature space.Finally, I used a LightGBM regression model to perform predictions.

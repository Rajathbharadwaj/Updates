One glaring example was that I realized a user profile was impacted by the size of the user dataset. If I was doing a project where I would just be presenting results, I would want to get as much clean data as possible. Therefore, it would reflect how much a user liked a given attribute relative to that whole dataset. In the article, the author uses one big dataset that has both unknown articles and articles the user has engaged with. Additionally, I encountered an issue where some pattern codes would throw an error, and that error would mess up the whole batch of patterns being processed. The solution I came up with in that issue was to divide each user profile number by the length of the dataset. This way, a user’s numbers wouldn’t differ just because of how much they used, or didn’t use, Ravelry for their projects and favorites.
